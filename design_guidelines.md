# Дизайн-гайдлайны: Система управления документацией газопроводов

## Подход к дизайну

**Выбранная система:** Carbon Design System (IBM)

**Обоснование:** Enterprise-система для управления критической инфраструктурой требует проверенных паттернов для работы с большими объемами данных, сложными таблицами и многоуровневыми интерфейсами. Carbon специализируется на промышленных и корпоративных приложениях с акцентом на эффективность и масштабируемость.

## Типографика

**Шрифты:** IBM Plex Sans (основной), IBM Plex Mono (код/технические данные)

**Иерархия:**
- H1: 32px, weight 400 (заголовки модулей)
- H2: 24px, weight 400 (секции панелей)
- H3: 20px, weight 600 (карточки, диалоги)
- Body: 14px, weight 400 (основной текст)
- Caption: 12px, weight 400 (метаданные, подписи)
- Code/Technical: 13px, IBM Plex Mono (ID объектов, коды)

## Система отступов

**Базовые единицы Tailwind:** 2, 4, 6, 8, 12, 16

**Применение:**
- Внутренние отступы карточек: p-6
- Отступы между элементами форм: space-y-4
- Отступы секций: py-8
- Компактные элементы (кнопки, теги): px-4 py-2
- Таблицы: p-4 для ячеек

## Структура layout

**Главный layout:**
- Фиксированная боковая навигация слева (256px)
- Хедер с breadcrumbs и действиями пользователя (64px высота)
- Рабочая область с max-width: none (использует всё доступное пространство)
- Контейнеры внутри: px-8 py-6

**Модальные окна и диалоги:**
- Стандартная ширина: max-w-2xl
- Широкие формы: max-w-4xl
- Полноэкранные модули (QR-сканер): w-screen h-screen

## Библиотека компонентов

### Навигация
**Боковая панель:**
- Вертикальное меню с иконками и текстом
- Сворачиваемые секции для модулей
- Индикатор активной страницы
- Счетчики уведомлений (badges)

**Breadcrumbs:**
- Горизонтальная навигация в хедере
- Разделитель: "/" 
- Последний элемент не является ссылкой

### Формы
**Поля ввода:**
- Высота: h-10
- Полная ширина контейнера
- Лейблы над полями (mb-2)
- Обязательные поля с астериском (*)
- Сообщения об ошибках под полем

**Мастер добавления объектов:**
- Пошаговый индикатор вверху
- Кнопки "Назад"/"Далее"/"Готово" внизу
- Каждый шаг занимает полную высоту

**Конструктор прав:**
- Чекбоксы сгруппированы по модулям
- Визуальные разделители между группами
- Поиск по правам

### Отображение данных

**Таблицы:**
- Полная ширина контейнера
- Фиксированный хедер при скролле
- Чередующиеся строки для читаемости
- Действия в последней колонке (иконки)
- Inline-редактирование где возможно
- Пагинация внизу

**Карточки документов:**
- Grid layout: grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4
- Превью документа (иконка типа файла)
- Название, дата, размер
- Теги категорий
- Действия при hover

**Панель аудита:**
- Временная шкала слева
- События справа
- Фильтры сверху (дата, пользователь, действие)
- Бесконечный скролл

### Специализированные компоненты

**Организационная структура:**
- Древовидная структура с отступами
- Иконки раскрытия/скрытия
- Drag-and-drop для реорганизации
- Контекстное меню на правый клик

**QR-сканер:**
- Полноэкранный интерфейс
- Видео-превью камеры в центре
- Результат сканирования: модальное окно с данными объекта
- Кнопка "Закрыть" в углу

**Просмотрщик документов:**
- Toolbar вверху (скачать, печать, версии)
- Iframe или canvas для отображения
- Боковая панель с метаданными (справа, сворачиваемая)

**Модуль обучения:**
- Список программ: grid-cols-1 lg:grid-cols-2
- Карточка программы: превью видео, название, прогресс
- Детальная страница: видеоплеер, материалы, тесты, сертификат
- Тесты: вопрос-ответ с индикатором прогресса

### Оверлеи

**Уведомления (Toast):**
- Позиция: top-right
- Стек уведомлений (max 3 одновременно)
- Автозакрытие через 5 секунд
- Типы: success, error, warning, info

**Подтверждения:**
- Центрированное модальное окно
- max-w-md
- Заголовок, описание, две кнопки

## Адаптивность

**Брейкпоинты:**
- Mobile: до 768px - одна колонка, скрытое меню (гамбургер)
- Tablet: 768px-1024px - две колонки для карточек, видимое меню
- Desktop: 1024px+ - полный функционал

**Мобильные особенности:**
- Боковое меню превращается в выдвижную панель
- Таблицы трансформируются в карточки
- QR-сканер использует нативную камеру
- Упрощенные формы (меньше полей на экране)

## Анимации

**Минималистичные переходы:**
- Открытие модальных окон: fade + scale (200ms)
- Раскрытие меню: slide-down (150ms)
- Загрузка: спиннер или скелетон
- **Избегать:** сложных анимаций при скролле, параллакса

## Ключевые принципы

1. **Информационная плотность:** максимум данных при сохранении читаемости
2. **Последовательность:** единые паттерны во всех модулях
3. **Доступность:** контрастность текста, поддержка клавиатуры, ARIA-метки
4. **Производительность:** ленивая загрузка таблиц, виртуализация списков
5. **Безопасность визуальная:** индикаторы уровня доступа (badges), предупреждения при критических действиях
